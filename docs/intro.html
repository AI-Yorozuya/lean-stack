資料模型 × 使用者動作 model × action

四個「東西」、三條關聯。會員下單；訂單含明細；明細下單當下從商品抄快照（之後改目錄不動歷史）。

會員 Member 1 — N 訂單
name email* phone status
動作　看清單 · 搜尋（新增/編輯/刪除＝你試缺口）
商品 Product 1 — N 明細
sku* name unit_price is_active
動作　看 · 搜尋 · 篩選 · 新增 · 編輯（inline 上架 / 加刪除＝練習）
訂單 Order 含 N 明細
order_no* member status total（衍生）
動作　新增 · 編輯 · 刪除 · 看詳細 ＋ 狀態機（收款/出貨/取消）
明細 OrderItem 快照 from 商品
name（快照）unit_price（快照）quantity subtotal（衍生）
鐵則　總額 = 明細加總 · 已出貨不可改
＊ = 業務識別碼（唯一，非主鍵）　·　衍生 = 後端算、不手填　·　快照 = 下單當下抄一份
訂單生命週期（狀態機：有向、有守衛、有終態）
待付款 收款→ 待出貨 出貨→ 已出貨
待付款 / 待出貨 —（取消）→ 已取消　·　非法轉移後端擋（422）
