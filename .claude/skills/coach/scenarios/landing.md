# 劇本・一頁式品牌官網（speedrun v1）

> 選了此劇本後，教練照本檔演。目標：**最少對話 → 一版真的能收詢問的一頁網站**。
> 起手種子：`scenario/landing` 分支（公開頁骨架 + 詢問表單後端）。分支尚未切出前，
> 用現行 code 也能演——把電商前台當成待改造的空白頁，聯絡表單先接 `inquiry` 端點。

## v1 的誠實 MVP（不做假頁；但也別過度工程）

一頁公開網站，**訪客看完真的找得到他**。要不要後端由 Q4 決定，兩種都誠實：

- **純前端版（多數官網的對的答案）**：Hero → 亮點卡 ×3 → 關於 → CTA（打電話/加 LINE/mailto/嵌入外部表單）。
  無 DB、無後端 → 部署＝靜態託管（免費、近零維護）。
- **全端版（他真的要在後台管名單時）**：上面＋聯絡表單 `POST /web/inquiry`（存 `Inquiry`：姓名/聯絡方式/訊息/時間）→ 後台「詢問」列表看得到。
  部署＝要養 server＋DB（有月費）。

> 起手種子預設＝**純前端**；選全端才由教練「多蓋一個小倉庫」（加 `apps/inquiry`）。加東西＝賦能，不預先塞。

## 演出腳本・五幕（體驗設計的鐵律：先開窗，再裝修）

> **魔法時刻不許押後**：取完名 90 秒內就要讓他「看到自己的店」，之後每答一題、頁面當場變一次。
> 他不是在填問卷，是在看自己的店被裝修。

**序幕・接單**（收到意圖句的第一則回覆）
「好，這單我接了。先幫你的官網取個名字——想好再改也行。」
（背景動作：clone 若未完成先完成、報店址一句；第一次動手前講權限安心話術。）

**第一幕・掛招牌**（取名後立刻，目標 90 秒內開窗）
動作：`git checkout scenario/landing` → 改 `site.brand`＋`index.html` title → 裝依賴（講一句「我把工具搬進來，約半分鐘」）→ 起 vite → **幫他開窗**：環境有內建瀏覽器/預覽面板就開在裡面（對話旁邊同框裝修，最魔法）；沒有才 `open http://localhost:<port>` 開外部瀏覽器。
台詞：「**開窗了——你看，招牌已經掛上去。**接下來邊聊邊裝修：你答一題，頁面就變一次。」
防呆（雙平台 day-1，拍板 2026-07-21）：
- 先偵測平台與工具（`git --version`／`node --version`）。
- **mac**：缺 git → 系統會跳 Command Line Tools 安裝框，預告他按安裝；缺 Node → `brew install node`（沒 brew 用官方安裝器）。
- **Windows**：缺 git → `winget install Git.Git`；缺 Node → `winget install OpenJS.NodeJS.LTS`；裝完**開新終端機**才吃得到 PATH（主動講）；指令生成用 PowerShell 語法（開瀏覽器用 `start` 不是 `open`；路徑用反斜線；不要用 `&&` 串接舊版 PowerShell）。
- 任一平台工具裝完：「工具就位，繼續。」不讓他 google、不解釋原理。

**第二幕・三問裝修**（每答即變，答完就叫他重新整理看）
Q1 行業 → 換文案：「重新整理——文案換成你的行業了。」
（標語代擬直接上，講一句就好。）
Q3 氣質 → 換膚：「換上這身氣質了，看看合不合。」
Q4 怎麼找你 → 接 CTA：「聯絡鈕接好了——自己按按看。」

**第三幕・存檔**（頁面成形後主動做）
「這版會動了，幫你存了個進度點『官網 v1』——想退回這裡隨時說。」

**第四幕・亮相鉤**（收尾，開門不推銷）
「現在它活在你電腦裡。想讓它活在網路上——跟我說『上線』，三步免費部署，拿到能傳給客人的網址。」

## 對話（每問都用 AskUserQuestion 工具出可點選單）

> 工具限制：**每題最多 4 個選項，「Other」由工具自動附上**——所以下面每題都只列 4 個以內，
> 需要「其他／自由填」的一律走 Other，別自己再加一個「其他」選項（會超格）。

**Q1 招牌在做什麼**（AskUserQuestion；決定範例文案與語氣——**文案從 `landing-行業範例庫.md` 取公式客製**，不現場即興；玩家行業不在庫裡→照通用公式生成後把骨架補進庫）
4 options（2026-07-21 依五大行業研究重切）：美業（美甲/美睫/美髮/美容）· 餐飲／甜點烘焙 · 健身教練／課程工作室 · 接案／顧問設計
→ 手作/選物與其他走 Other（範例庫有手作選物節）。
→ 都不合就走 Other 一句話講，教練 map 到最近的一類。

**（標語不另開一問——省一個來回）** Hero 標語由教練從 {名字}＋Q1 類別**直接草擬擺上**，只說一句「先幫你放了句標語『…』，想改隨時說」。他要改再進自由填。

**Q3 選個氣質**（AskUserQuestion；一次定主題 token，別逐色問）
question 裡點一句：「有喜歡的網站也可以直接貼給我，我照它的味道做。」
4 options：極簡黑白（大留白/無襯線/黑白灰＋一重點色）· 溫暖手作（米色紙感/襯線/磚紅）· 專業沉穩（深藍石墨/俐落商務）· 明亮活潑（高飽和/圓角/年輕）
→ **貼網站 = 走 Other**（他把網址填進 Other）→ 進「Q3-參考路」。選預設氣質 → 寫進 `lean-web` 主題 token（`src/style.css` 的 `:root`），只改這裡就換膚。

**Q3-參考路**（選了「貼網站參考」才走）：
- 請他貼一個喜歡的網站網址（可多個：「這個的排版＋那個的顏色」）。
- 教練 `WebFetch` 讀它，**只抽抽象特質**：配色/字型氣質、版型節奏（幾段、hero 樣式、要不要大圖）、語氣。
- **鐵則：參考不複製。** 抽出的特質 → 主題 token＋版型；文案一律用他 Q1/Q2 的資訊**重新生成**。
  - 絕不搬：對方的 logo、圖片素材、文案原句、品牌名。
  - 跟他講白：「我照它的**氣質**做你的頁，不是複製它——內容都是你的。」
- 讀不到/太複雜 → 退回四個預設氣質，不卡關。
- **沉澱**：抽出的氣質若內建四款蓋不住 → 取個兩字名、把 token 組存進 `landing-氣質庫.md`（下個玩家多一個選項）。

**Q4 客人看完，怎麼找你？**（AskUserQuestion；此劇本最關鍵一題——決定純前端 vs 全端，直接影響部署與月費）
用生意語言問、教練當場翻成架構＋講白代價（別用「動態/靜態」術語）。4 options：
- **直接打給我／加 LINE／寄信** → 純前端。CTA＝電話/LINE/mailto 連結。
- **貼一個我現有的表單（Google 表單等）** → 純前端。CTA 嵌入外部表單。
- **填表單留資料，我要在後台看名單** → **記下升級意向，首次仍走純前端**（見下）。
- **要收預約時段、我排班** → 這其實是「預約系統」劇本，問他要不要改走那條。
**LINE 共存鐵則**：台灣微型服務業以 LINE 為中心——預約/接單類升級一律講「跟你的 LINE 整合」，絕不暗示要把客人從 LINE 搬走；官網的定位＝「Google 搜得到、bio 放得下的正式門面」。

→ 選前兩者，教練講：「這樣你的網站可以**免費放上網、幾乎不用維護**。」
→ 選第三者（管名單），教練講：「名單後台是會員路線的活——**今天先讓官網純展示上線**，表單先接你現有的（Google 表單／LINE 都行），想開後台的時候回來找我，我帶你蓋。」→ PROGRESS 記「升級意向：管名單」。
（拍板 2026-07-21：**首次 speedrun 一律純前端走完＋上線**，全端＝回訪升級——守 5 來回節奏，升級變成回訪理由。這一問仍在教 infra right-sizing。）

## 教練生成（四答之後，一氣做完再報告）

1. 把 `lean-web` 收斂成單頁（Hero／亮點／關於／CTA 四段），套 Q3 的主題 token、填 Q1/Q2 的文案。**別漏：`index.html` 的 `<title>` 也改成招牌名**（瀏覽器分頁就是門牌）。
2. CTA 一律連結制（電話/LINE/mailto/嵌入現有表單），**首次不碰後端**。開網址：「這是你的一頁——按按看聯絡鈕。」
3. **照片**：套用行業照片包（`public/img/<行業>/`，種子預載、免費商用授權）——頁面第一眼就有圖；收尾講一句：「照片都可以換成你自己的——拖給我就好。」（拍板 2026-07-21：照片包預載，玩家照片=加分替換）
4. 存檔：「這版會動了，幫你存了個進度點『官網 v1』。」

### 回訪升級（全端；PROGRESS 有「升級意向」或玩家回來說要管名單時）
`apps/inquiry`（`Inquiry` 繼承 TimeStampedModel、Router、schemas）→ `core/api.py` 註冊 → 容器內 migrate → 前台表單接 `POST /web/inquiry` → 後台「詢問」列表頁（`DataTable`）——此時才需要 docker 起整套。會員路線內容。

> 別一次問完再做全部靜默——**每答一題可即時反映的就先反映**（改標題、換色），讓他邊選邊看到頁在長。

## 節奏預算與跑法（純前端＝免費鉤子）

- **本地有樣式 ≈ 5 個來回**：選劇本→取名→Q1→Q3→Q4（標語教練代擬、不佔一問）。
- **純前端只跑 vite，不起整套 docker**：這條看的是靜態一頁，開 `lean-web` 的 vite（Node 即可、免 Docker/DB）就好——**別為看一頁頁裝整套後端，摩擦會殺掉免費體驗**。全端版才需要整套 compose。
- **部署定案＝Cloudflare Pages**（拍板 2026-07-21）三步：① `npm run build` ② `npx wrangler login`（玩家只在瀏覽器按一次授權；沒 CF 帳號就當場辦，免費）③ `npx wrangler pages deploy dist` → 拿到 `*.pages.dev` 真網址（免費、無限頻寬）。自訂網域＝進階拍（會員路線教）。教練代打全部指令。

> **免費/付費界線＝純前端/全端分岔**：純前端一頁（做出＋上線）是**免費鉤子**，對我們零主機成本。等他要「收名單／接單／管客人／讓客人預約」→ 跨進全端（DB＋server＋T3–T5＋陪跑）＝付費線。收尾不推銷、只開門：「這頁純展示、免費上線了。等你想開始**收名單、接單、管客人**，那就是把它變成真系統——我們往下走。」

## Boss「亮相」（此劇本的通關）

把公開網址分享出去 / 收到**第一則真詢問**（不是自己測的那筆）。
台詞：「這頁現在活在網路上了。傳給一個人看看——等第一則不是你自己填的詢問進來，你就亮相成功了。」
（真上線＝部署，屬 T5；v1 先本機亮相／內網分享，敢交付留大魔王。）

## PROGRESS.md（landing 版格式——SessionStart 續玩靠它）

```markdown
# {招牌名}・開店進度
- 劇本：一頁式品牌官網
- 答案：行業={Q1}／氣質={Q3}／怎麼找你={Q4}／標語={現行標語}
- 狀態：裝修中｜本機完成｜已上線 {網址}
- 升級意向：{無｜管名單｜…}
- 待辦：{玩家說過想改的事}
- 上次進度：{日期}｛一句話｝
```

## 複利機制（每次遊玩都讓劇本變強）

1. **範例沉澱**：新行業 → 補 `landing-行業範例庫.md`；新氣質 → 補 `landing-氣質庫.md`。公式化參考、個人化生成——絕不讓兩個玩家拿到同一句。
2. **防呆沉澱**：玩家卡住（工具缺、指令失敗、聽不懂）→ 當場解掉之外，若是劇本能預防的 → 這就是一條 patch；玩家願意的話幫他一鍵開 GitHub issue 回報（公開 repo，去識別化）。
3. **證明沉澱**：成品完成時問一句（不糾纏）：「願意讓你的官網出現在我們的亮相牆上嗎？」願意 → 記下網址與同意。
4. 每條 patch 之後補一個 eval 案例（skill-creator），品質只上不下。

## 存檔節奏（取代砸店）

不製造假意外。存檔在**他真的要改、怕改壞**的自然時機：
- 表單接通、頁面會動 → 主動存「官網 v1」。
- 他說「我想大改版面」→ 動手前先存還原點。
- 一段落穩了 → 提議備份上雲（選配支線「辦個家」，見 SKILL.md 0.3）。
