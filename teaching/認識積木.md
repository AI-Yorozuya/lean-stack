## 上半 · 表層：會員/商品列表頁（學會「指名」）

### S03 · 一頁 ＝ 完整增刪查改（七元件四 API）★核心
- **目標**：看穿「列表頁 ＝ 幾個元件 × 四個 API」，每個元件都因一個動作而存在。
- **畫面**：截列表頁（`screens/s03-list.png`），把元件框起來標號（就是「把 input/select/button/table/pagination 都框起來」）。
- **標註**：① table＋pagination ② input＋select＋button ③ 新增 button→dialog＋form ④ 每列 編輯/刪除。
- **動作 → 元件 → API**（正本表，逐字取自 `認識積木_v0`）：

  | 你做的事 | 用到的元件 | 背後的 API |
  |---|---|---|
  | 看全部訂單 | table + pagination | 讀清單 |
  | 篩選 | input + select + button | 讀清單（帶條件） |
  | 新增 | button → dialog + form | 建立 |
  | 編輯 | button → dialog + form | 修改 |
  | 刪除 | button → dialog 確認 | 刪除 |

  → **七個元件、四個 API，全在一頁**。管客戶/商品/員工——同一套，換個名詞。（表層＝會員/商品，就是這套。）
- **講稿重點**：① 元件**保留英文**（table/input/select…）＝指名的價值，講 AI 認得的詞。② 本 sandbox 的會員/商品「刪除」用**停用不刪**（軟刪），硬刪的「為什麼不刪」留到底層講。
- **對應**：lean-stack `/members`、`/products`（表層）、`積木詞彙對照` §三

### S04 · 指名練習：把整句講給 AI
- **目標**：示範精準指名（worked example）——不只指名「頁面」，也能指名「互動」。
- **畫面**：一句 prompt ＋ AI 一次做對的結果。
- **標註**：—
- **講稿重點**：
  ① **指名一個頁面**——照唸：「做一個商品列表頁：table 列出、pagination 分頁，上面用 input 和 select 篩選；每列能編輯和刪除，新增用 dialog 表單。」它一次給對，因為每個字它都認得。
  ② **指名一個互動（inline 修改）**——看**商品**的「上架狀態」欄現在是**純文字**、只能看。你想不進編輯頁、**直接在列表切**上架/下架，怎麼講？「把上架狀態做成可以**直接在列表切換**的開關（switch）。」← 一句話同時指名了新元件 `switch` ＋「**inline 修改**」這個互動。
  ③ **指名一個動作（加刪除）**——商品的操作欄現在只有編輯。你想加刪除，怎麼講？「操作欄加一個**刪除**按鈕，按了跳 dialog 確認再刪。」← 用 `button`＋`dialog` 組出「刪除（DELETE）」這個 API 動作。
- **參考答案**：switch 不在核心七元件裡——是「你已經會七個、再加一個新積木」的加分題（sandbox 早期版就這樣做，git `65f367a`）。缺 switch 就 `npx shadcn-vue add switch`；刪除按下要接後端 DELETE。
- **對應**：`認識積木_v0`「練習：指名下單」；lean-stack `/products`（上架＝純文字、操作只有編輯 ＝ 兩個缺口）

## 下半 · 底層：訂單詳細頁（學會「判斷」）

### S06 · 三問：看一張訂單詳細頁 ★核心
- **目標**：拿到判斷 AI 規劃的 checklist。
- **畫面**：截訂單詳細頁 `/orders/:id`（`screens/s06-detail.png`），三問各指一塊。
- **標註**：框「會員/明細/金額」「狀態列」「總計」三處。
- **三問**（正本表）：

  | 問一句 | 看頁面哪裡 | 這叫什麼 |
  |---|---|---|
  | 有什麼？跟誰有關？ | 會員、明細、金額 | 資料模型 |
  | 能發生什麼？什麼順序？ | 待付款 → 待出貨 → 已出貨 / 已取消 | 狀態與流程 |
  | 什麼必須永遠成立？ | 總額=明細加總、已出貨不能改 | 鐵則 |

- **講稿重點**：這三句就是判斷 checklist。三問 ＝ **承重牆的大眾裁剪**（資料 × 狀態 × 不變量）。
- **對應**：lean-stack `/orders/:id`、`intents/訂單管理.md`、`積木詞彙對照` §四
